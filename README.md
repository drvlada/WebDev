# HealthPlate

Виконали: Драган Влада, Завальська Анастасія КІ-33

## Опис проєкту
HealthPlate - це онлайн-сервіс для персоналізованого планування здорового харчування. Користувачі можуть створювати індивідуальні плани харчування, враховуючи вік, стать, рівень активності, цілі, харчові вподобання та алергії. Сервіс надаватиме готові рецепти, підрахунок калорій та формування списку продуктів.

## Основний функціонал 
* Можливість зареєструватись, увійти в профіль, а також редагувати його
* Перегляд стрічки зі статтями
* Пошук рецептів за фільтрами, перегляд їх харчової цінності, зберігання улюблених рецептів
* Створення персоналізованого меню на основі особистих параметрів та мети (зменшення/утримання/набір ваги)
* Можливість зворотнього зв'язку

## Основні сторінки
* index.html - головна сторінка, опис функціоналу сервісу
* about.html - інформація про розробників
* authorization.html - авторизація користувача
* community.html - стрічка, у якій публікуються статті пов'язані зі здоров'ям та їжею
* contacts.html - сторінка з контактами для зворотного зв'язку
* menu.html - створення персоналізованого меню
* profile.html - профіль користувача
* recipes.html - перелік рецептів за вподобаннями

## Використані технології
* DOM 
Технологія DOM використовується для динамічного керування вмістом вебсторінок. Вона дозволяє змінювати структуру HTML-документа без перезавантаження сторінки, додавати або видаляти елементи інтерфейсу, оновлювати текстову інформацію та атрибути елементів. Завдяки DOM у проєкті реалізовано динамічне відображення списків рецептів, статей спільноти, сторінок окремих рецептів і повідомлень для користувача, а також зміну зовнішнього вигляду елементів інтерфейсу залежно від дій користувача.
* Events 
Механізм обробки подій застосовується для реагування вебсторінок на дії користувача. У проєкті він використовується для обробки кліків, наведення курсора, відправки форм, вибору файлів та натискання клавіш. Саме завдяки подіям реалізовано роботу мобільного меню, перемикання теми оформлення, додавання рецептів до улюблених, взаємодію з формами, а також навігацію між сторінками. Подієва модель забезпечує інтерактивність сайту та миттєвий зворотний зв’язок для користувача.
* Validation
Валідація використовується для перевірки коректності даних, які вводить користувач. У проєкті вона застосовується перед відправленням форм, зокрема під час реєстрації, входу в систему, редагування профілю та надсилання контактних повідомлень. Валідація дозволяє запобігти надсиланню порожніх або некоректних даних, підвищує надійність роботи сайту та покращує користувацький досвід, оскільки помилки виявляються безпосередньо на стороні клієнта.
* LocalStorage
LocalStorage використовується для збереження стану вебзастосунку безпосередньо в браузері користувача. У проєкті ця технологія застосовується для збереження користувацьких налаштувань та тимчасових даних, таких як обрана тема оформлення, список улюблених рецептів, параметри інтерфейсу та інформація про авторизованого користувача. Використання LocalStorage дозволяє зберігати ці дані між сесіями та забезпечує персоналізований досвід без необхідності повторного введення інформації.
* Fetch API
Fetch API використовується для асинхронної взаємодії між клієнтською частиною сайту та джерелом даних. Завдяки цій технології сторінки можуть отримувати та надсилати інформацію у фоновому режимі без перезавантаження. У проєкті Fetch API застосовується для завантаження рецептів, статей спільноти, даних профілю користувача та результатів роботи адмін-панелі. Це забезпечує актуальність контенту та плавну роботу інтерфейсу.
